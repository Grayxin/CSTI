# CSTI（公务员性格测试）详细评分与结算标准体系

为了让这套系统能够被精准地开发成小程序或 H5 测试工具，以下是为你量身定制的 CSTI 详细评分标准。

本套评分系统采用了**“绝对值对抗+极端阈值熔断”**的双层计算逻辑，既能测出常态化的体制内人格，也能精准捕捉那些已经“异化”的突变体。

---

## 一、 基础计分模型：双轨标签制

在 30 道题目中，每个选项都绑定了 **2 个** 维度标签（例如：选 A 获得 `+1 O` 和 `+1 P`）。
测试完成后，系统将在后台统计用户在 8 个字母上的累计得分。

### 1. 计分池（最高理论总分 60 分）
系统包含 4 个相互对抗的计分池：
* **【精力代谢池】**：`L`（满载） vs `D`（枯竭）
* **【社交厚度池】**：`O`（圆滑） vs `S`（耿直）
* **【情绪水位池】**：`N`（麻木） vs `F`（易碎）
* **【权力感知池】**：`P`（入世） vs `A`（出世）

---

## 二、 常规性格结算规则（16 种基础人格）

这是最基础的结算逻辑，适用于 90% 的测试者。

### 1. 维度对抗判定（胜者通吃）
系统将每个计分池中的两个字母进行对抗，得分高者成为最终的性格代码。
* **逻辑示例**：如果用户的得分为 `L: 8, D: 4`，则该维度结果为 **L**。
* 最终将 4 个胜出的字母组合，生成四字代码（如：`LONP`、`DSFA`），并在【16种基础人格库】中调取对应的结果卡片。

### 2. 同分处理机制（“体制内中庸”平局规则）
如果在某个维度出现平局（例如：`L: 6, D: 6`），说明该员工在该维度处于“薛定谔状态”。系统采取以下**默认降级原则**（因为在体制内，不主动就是退缩）：
* 平局 `L = D` -> 强制判定为 **D**（枯竭：既然不卷，那就是想躺）
* 平局 `O = S` -> 强制判定为 **O**（圆滑：能活到现在，多少沾点圆滑）
* 平局 `N = F` -> 强制判定为 **N**（麻木：真正的易碎早就辞职了，留下的都是麻木）
* 平局 `P = A` -> 强制判定为 **A**（出世：升职这事儿，不争就是不要）

## 三、 突变人格触发机制（9 种极端人格）

为了增加测试的“疯感”和社交分享的爆发力，我们引入了**阈值熔断机制**。当某个维度的得分高得离谱时，将直接无视其他三个维度，强制触发【极端突变人格】。

### 1. 极端阈值设定
假设单维度理论最高得分为 15 分。
* **触发条件**：某一字母单项得分 >= **12 分**（即该维度占比达到 80% 以上）。

### 2. 突变触发优先级（防冲突机制）
如果一个用户精神状态极度不稳定，同时有两个或以上的字母得分 >= 12，系统按以下优先级强制结算（按体制内生存杀伤力排序）：

* **第一优先级：权力与情绪爆表**
    * 如果 `P` >= 12 -> 触发 **【权力多巴胺成瘾者】**
    * 如果 `F` >= 12 -> 触发 **【尖叫的土拨鼠】**
* **第二优先级：社交属性爆表**
    * 如果 `O` >= 12 -> 触发 **【人形变色龙】**
    * 如果 `S` >= 12 -> 触发 **【职场泥石流】**
* **第三优先级：精力与麻木爆表**
    * 如果 `L` >= 12 -> 触发 **【终极八爪鱼】**
    * 如果 `D` >= 12 -> 触发 **【量子态员工】**
    * 如果 `N` >= 12 -> 触发 **【三体人降临】**
    * 如果 `A` >= 12 -> 触发 **【修仙者】**

## 四、 隐藏彩蛋触发条件

为了让产品更有话题性，必须设置极小概率的隐藏彩蛋结果。

**彩蛋：【薛定谔的临时工】**
* **触发条件**：只有当用户的四个维度全部触发平局规则，即测试结果为绝对均衡的（例如 `L=D` 且 `O=S` 且 `N=F` 且 `P=A`）。
* **开发逻辑**：检测所有对抗池的差值是否都 <= 1。如果是，则不走基础人格结算，直接弹出此隐藏卡片。
* **文案提示**：“系统无法定位你的存在。你既在编制内，又在编制外；你既是中流砥柱，又是背锅首选。你触发了体制内的最高机密——薛定谔的临时工。”

---

### 结算流程伪代码：
1. `let scores = { L:0, D:0, O:0, S:0, N:0, F:0, P:0, A:0 };`
2. 遍历用户所有选择，将 `scores` 对应累加。
3. **Step 1 检查彩蛋**：如果 `abs(L-D)<=1 && abs(O-S)<=1...` -> 输出【薛定谔的临时工】。
4. **Step 2 检查突变**：按照优先级 `P -> F -> O -> S -> L -> D -> N -> A` 检查是否有得分 >= 12 的维度。如果有，立刻 `break` 输出极端人格。
5. **Step 3 常规结算**：对比 `L vs D`, `O vs S`, `N vs F`, `P vs A`。拼接四位字母字符串（如 "LONP"）。
6. 根据字符串在数据库中匹配结果卡片。
